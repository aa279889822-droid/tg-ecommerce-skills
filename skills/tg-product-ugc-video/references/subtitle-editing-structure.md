# Subtitle and Editing Structure

Use this reference when producing editing scripts and subtitle structures.

## Editing Table

Use this table:

| Time | Clip | Visual Action | Voiceover | On-screen Text | Edit Notes |
| --- | --- | --- | --- | --- | --- |

Guidelines:

- First 1-3 seconds must carry the hook.
- Show product within the first 3 seconds unless the chosen style needs a very short setup.
- Keep each shot simple enough to generate or edit.
- Use text overlays for hooks, proof points, and CTA.
- Avoid filling the screen with long text.

## 15-Second Structure

Use 4-6 shots:

1. 0-2s: Hook
2. 2-5s: Problem or desire
3. 5-9s: Product demo
4. 9-12s: Result or reason to believe
5. 12-15s: CTA

## 30-Second Structure

Use 6-9 shots:

1. 0-3s: Hook
2. 3-6s: Problem/context
3. 6-10s: Product reveal
4. 10-16s: Demo
5. 16-21s: Benefit/proof
6. 21-26s: Objection handling
7. 26-30s: CTA

## 45-Second Structure

Use 8-12 shots:

1. 0-3s: Hook
2. 3-7s: Relatable context
3. 7-12s: Problem detail
4. 12-17s: Product reveal
5. 17-25s: Demo steps
6. 25-32s: Result or comparison
7. 32-38s: Trust detail or objection handling
8. 38-45s: CTA

## SRT Format

Use this format:

```srt
1
00:00:00,000 --> 00:00:02,000
Subtitle line here

2
00:00:02,000 --> 00:00:05,000
Subtitle line here
```

Rules:

- Keep each subtitle short and spoken.
- Split long lines into multiple captions.
- Match subtitle timing to the shot list.
- Keep on-screen text separate from subtitles.

## On-Screen Text Rules

Use on-screen text for:

- Hook
- Pain point
- Key benefit
- Comparison label
- CTA

Avoid:

- Full paragraph overlays
- Claims that are stronger than the voiceover
- Tiny text that cannot be read on mobile

## Export Notes

Default export:

- Aspect ratio: 9:16
- Resolution: 1080x1920
- Captions: burned-in for TikTok/Reels/Douyin; optional SRT for YouTube Shorts
- Safe area: keep important text away from bottom-right platform UI area
